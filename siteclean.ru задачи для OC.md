Вот мои точные требования для максимальной эффективности сайта:

### 1. Дизайн и брендинг
- **Цветовая палитра:** 
  - Фон: Чистый белый (#FFFFFF) и очень светлый серо-голубой (#F8FAFC) для разделения секций.
  - Основной текст: Глубокий темно-синий/графит (#0F172A) для максимальной читаемости.
  - Акцентный цвет (Кнопки CTA, иконки): Яркий, доверительный синий (#2563EB) или изумрудно-зеленый (#10B981), ассоциирующийся с "успехом" и "чистотой". Выбери изумрудно-зеленый для главных кнопок действия.
- **Шрифты:** Подключи через Google Fonts: `Inter` для основного текста (максимальная читаемость) и `Manrope` для заголовков (современный, технологичный вид).
- **Стиль:** "Кибер-чистота". Минимализм, много "воздуха" (whitespace), мягкие тени у карточек (box-shadow), скругленные углы (border-radius: 8px или 12px). Никакого визуального шума.

### 2. Контент и тексты (Ключевые сообщения)
- **Hero-блок (H1):** "Экстренная очистка сайтов от вирусов за 24 часа"

- **Приоритетные CMS для упоминания:** WordPress (самый частый), 1С-Битрикс (самый дорогой B2B сегмент), OpenCart, Joomla.
- **Триггеры доверия (разместить в шапке или под H1):** "Работаем с юрлицами", "Гарантия 90 дней", "NDA".

### 3. Поведение формы и UX (Критически важно)
- **Поля формы:** 
  1. Имя (опционально, placeholder: "Ваше имя").
  2. Телефон (ОБЯЗАТЕЛЬНО, required). Нужна JS-маска ввода в формате `+7 (___) ___-__-__`.
  3. Адрес сайта (ОПЦИОНАЛЬНО, но желательно, placeholder: "https://...").
  4. Описание проблемы (ОБЯЗАТЕЛЬНО, required, textarea, placeholder: "Кратко опишите проблему: редиректы, блокировка Google и т.д.").
- **Обработка отправки (AJAX):** НЕ делай редирект на стандартную страницу Formspree. Используй Fetch API в Vanilla JS. ### 4. Технические детали
- **Методология CSS:** Строго BEM (Block Element Modifier) для чистоты и поддерживаемости кода.
- **Анимации:** Только легкие и производительные. Используй `Intersection Observer` на чистом JS для плавного появления элементов (fade-in-up) при скролле. Никаких тяжелых библиотек типа GSAP.
- **Мобильное меню:** Бургер-меню должно плавно выезжать или появляться с эффектом `backdrop-filter: blur(10px)` для современного вида.
- **Изображения:** Все иконки должны быть в формате inline SVG (для скорости и четкости). Для картинок-заглушек используй атрибут `loading="lazy"`.
